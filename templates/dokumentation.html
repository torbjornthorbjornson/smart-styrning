🌟 Vision

Målet är att skapa världens bästa fastighetsstyrning. Systemet ska vara intelligent, automatiserat, energieffektivt och datadrivet.

✅ Delmål 1 – Funktioner som ska uppnås (vissa redan klara)

1. Elpris & väderprognos

  • Hämta elpris per timme (senare 15-minutersintervaller)
  • Hämta väderdata (temperatur, vind, symbol_code m.m.)
  • Spara data i en SQL-databas (klart)

2. Fastighetsregister

  • Databas med fastighetsbeteckning, temperaturprofil, driftbehov
  • Visa och redigera via webbsida

3. Temperaturanalys

  • Beräkna medeltemperatur för kommande 24 timmar
  • Avgör drifttimmar för varje fastighet utifrån temperaturprofil

4. Optimerad styrning

  • Välj billigaste elpristimmarna
  • Styr värmepumpen enbart under valda timmar

5. Geopositionering & SE-kod

  • Använd Lantmäteriets API för position från fastighetsbeteckning
  • Avgör SE-område (SE1-SE4), hämta rätt väderdata

🌐 Webbgränssnitt

  • Flask-baserad webbsida
  • Använder HTML, CSS, Chart.js
  • Visar elpris & väder (graf & tabell)
  • Symboler (SVG/emojis)
  • Medelvärden
  • Redigera fastigheter & profiler

⚙️ Teknikstack

  • Backend: Python (Flask), MariaDB, RPi.GPIO, pymodbus
  • Frontend: HTML, CSS, JS, Chart.js, Bootstrap
  • Server: Raspberry Pi 5, Gunicorn
  • Automation: systemd, inotifywait, Git

🔀 Flexibilitet & framtidssäkring

  • Redo för 15-minutersintervaller
  • Temperaturprofiler per fastighet
  • Förberett för moln, fjärrstyrning, m.m.

Detta är första steget – men grunden är redan solid.
